"Docker is an open source containerization platform that lets you package an application and all its dependencies into a standardized unit called a container.
The key insight is that a container is not a virtual machine. A VM runs its own full operating system with its own kernel — that is why VMs take gigabytes of space and minutes to start. A container shares the host machine's OS kernel and only packages what the application actually needs. That makes containers megabytes instead of gigabytes and they start in seconds.
There are three concepts to understand:
A Docker image is the blueprint — a read only template that defines the OS layer, installed software, environment variables, and application code. You build it once from a Dockerfile.
A Docker container is a running instance of that image. The same image can spawn ten containers simultaneously — like instantiating the same class ten times in Java.
Docker Compose is a tool for defining and running multi-container applications. Instead of manually starting each container with separate commands, you define all your services in a single docker-compose.yml file. In ResumeMaster I have two services — the Spring Boot backend and PostgreSQL. The backend declares depends_on: db so Docker Compose starts PostgreSQL first, then Spring Boot. They communicate over Docker's internal network using the service name as the hostname — Spring Boot connects to PostgreSQL at db:5432 instead of localhost. One command — docker-compose up -d — starts the entire stack in detached mode."
