"Nginx is a reverse proxy and web server that lives on my EC2 instance. It handles SSL termination — it intercepts all HTTPS traffic on port 443, decrypts it using the Let's Encrypt certificate, and forwards plain HTTP internally to Spring Boot on port 8080. Port 8080 is never exposed to the internet so Spring Boot is completely hidden. If someone asks why not just use Spring Boot directly — two reasons: Let's Encrypt certificates expire every 90 days and Certbot auto-renews them with zero downtime in Nginx, whereas Spring Boot would require a restart. And second, separation of concerns — Nginx handles network concerns, Spring Boot handles application concerns."
